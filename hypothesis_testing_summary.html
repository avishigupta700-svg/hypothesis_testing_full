§1

The Four Steps of Hypothesis Testing

01

State the Hypotheses

Write H₀ (null) and H₁ (alternative). H₀ is assumed true. H₁ states what you think is wrong about H₀.

02

Set the Criteria

Choose the level of significance (usually α = .05). This sets the rejection region using critical values.

03

Compute Test Statistic

Calculate zobt to find how many standard deviations the sample mean is from the null population mean.

04

Make a Decision

Compare zobt to critical value. If p < .05 → Reject H₀. If p > .05 → Retain H₀.

§2

Key Definitions

Null Hypothesis (H₀)
A statement about a population parameter assumed to be true. The starting point — we test whether it should be rejected.
Alternative Hypothesis (H₁)
Directly contradicts H₀. States the parameter is less than (<), greater than (>), or not equal (≠) to the null value.
Level of Significance (α)
The criterion for deciding whether to reject H₀. Typically set at .05 in behavioral research.
Test Statistic (zobt)
A formula that converts the sample mean into a z-score, telling how far it is from the population mean stated in H₀.
p value
The probability of obtaining a sample outcome given H₀ is true. Always between 0 and 1, never negative.
Statistical Significance
When H₀ is rejected (p < .05), the result is significant. When retained (p > .05), result is not significant.
Critical Value
Cutoff value(s) marking the boundary of the rejection region. If zobt exceeds this, reject H₀.
Rejection Region
The area beyond the critical value(s). Any zobt falling here → reject H₀.
Standard Error (σM)
σM = σ / √n. The standard deviation of the sampling distribution. Gets smaller as n increases.
§3

Types of Error in Decision Making

Decision: Retain H₀ Decision: Reject H₀
Truth: H₀ is TRUE ✓ CORRECT
1 − α		 ✗ TYPE I ERROR (α)
Rejecting a true null
Truth: H₀ is FALSE ✗ TYPE II ERROR (β)
Retaining a false null		 ✓ CORRECT — POWER
1 − β

Type I Error (α)

Rejecting a true H₀. Like convicting an innocent person. Researchers directly control this by setting α = .05.

Type II Error (β)

Retaining a false H₀. Like letting a guilty person go free. Associated with decisions to retain H₀.

Type III Error

Retaining H₀ because the rejection region was placed in the wrong tail. Only possible with one-tailed tests.

Memory tip: α is the "alarm you set" — you directly control it. β is what slips through when the alarm doesn't go off. Power = 1 − β = the probability you catch a real effect.
§4

One-Independent Sample z Test

Used when testing a population mean with a known variance (σ²).

z Statistic Formula
zobt = (M − μ) / σM    where    σM = σ / √n
M = sample mean  |  μ = population mean (from H₀)  |  σ = population SD  |  n = sample size
Worked Example (GRE scores): μ = 558, σ = 139, n = 100, M = 585
σM = 139 / √100 = 13.9  →  zobt = (585 − 558) / 13.9 = 1.94

Finding p value

  • Look up |zobt| in the unit normal table (column A)
  • Get the tail probability (column C) = p
  • One-tailed: p value = p (1×)
  • Two-tailed: p value = 2p (2×)

Making the Decision

  • If p < .05 → Reject H₀ ✓
  • If p > .05 → Retain H₀
  • If |zobt| exceeds critical value → Reject H₀
  • If |zobt| is within critical values → Retain H₀

Critical Values — Quick Reference

α Level One-Tailed (upper or lower) Two-Tailed (±)
α = .05 +1.645 or −1.645 ±1.96
α = .01 +2.33 or −2.33 ±2.58
α = .001 +3.09 or −3.09 ±3.30
For two-tailed tests: Split α in half — put α/2 in each tail.  |  For one-tailed tests: Put all α in one tail (upper or lower depending on H₁).
§6

One-Tailed vs. Two-Tailed Tests

Two-Tailed (≠)

H₁: μ ≠ value
Rejection region in both tails. More conservative. Eliminates Type III error. Most common in behavioral research.

Upper-Tail (>)

H₁: μ > value
Rejection region in the upper tail only. Use when outcome can only plausibly be above the null.

Lower-Tail (<)

H₁: μ < value
Rejection region in the lower tail only. Use when outcome can only plausibly be below the null.

One-Tailed Advantages

  • Greater power (easier to reject H₀)
  • Full α placed in one direction

One-Tailed Disadvantages

  • Risk of Type III error
  • Must justify directional prediction
  • Harder to publish if wrong direction
Key insight: The same data can give different decisions depending on tail direction. Example: zobt = 1.94 → Retain H₀ (two-tailed, α=.05) but Reject H₀ (upper-tail, α=.05). This is why Type III errors matter.
§7

Effect Size — Cohen's d

Hypothesis testing tells you whether an effect exists. Cohen's d tells you how big it is.

Cohen's d Formula
d = (M − μ) / σ
Uses population SD (σ) in denominator — NOT standard error (σM). This is the key difference from the z statistic.
Small
d < 0.2
Medium
0.2 ≤ d < 0.8
Large
d ≥ 0.8
GRE Example: d = (585 − 558) / 139 = 27/139 ≈ 0.19 → Small effect size.
This means scores shifted 0.19 standard deviations above the population mean.

Remember: Only report effect size when the result is significant (H₀ was rejected). Computing it for a retained H₀ is illogical — you'd be measuring the size of an effect you concluded doesn't exist.

z Statistic vs. Cohen's d

  • z uses σM (standard error) — affected by n
  • d uses σ (population SD) — not affected by n
  • z: "Is the effect real?" (p value)
  • d: "How big is the effect?"

What Affects Cohen's d?

  • ↑ Mean difference (M−μ) → ↑ d
  • ↑ Population SD (σ) → ↓ d
  • Sample size (n) has no effect on d
  • Larger d = more practically important effect
§8

Statistical Power

Power = 1 − β = probability of correctly rejecting a false H₀. Researchers aim for power ≥ .80.

↑ Increase to Boost Power

d (effect)↑Bigger true effect = easier to detect
n (sample)↑Larger n → smaller σM → bigger z
α (alpha)↑Larger rejection region (use with caution)

↓ Decrease to Boost Power

β (beta)↓Directly reduces Type II error rate
σ (pop SD)↓Less variability → cleaner signal
σM (SEM)↓Tighter sampling distribution
Effect size & power example: Two classes, same 2-point effect (M=40, μ=38), n=30:
Class 1 (σ=10) → d=0.20 → Power ≈ 29%  |  Class 2 (σ=2) → d=1.00 → Power ≈ 99.99%
Larger effect size = dramatically more power.
Sample size & power example: Class 1 (σ=10, μ=38, M=40):
n=30 → z = 1.10 → Retain H₀ (fails to reach critical value 1.645)
n=100 → z = 2.00 → Reject H₀ (exceeds critical value 1.645)
Increasing n alone changed the decision!
§9

APA Format — Reporting Results

What to Report

Always include: test statistic  p value  effect size  + a table or figure when possible.

"Test scores for students in the elite school were significantly higher than the standard performance of test takers, z = 1.94, p < .03 (d = 0.19)."

Language Rules

  • Say "significant" not "reject the null"
  • Say "not significant" not "retain the null"
  • Report exact p value when possible
  • Can also state p < closest round value (e.g., p < .03 for p = .0262)

Effect Size in Reports

  • Effect size is almost always reported with the test statistic
  • Significance testing alone cannot stand in research reports
  • Effect size can (e.g., in a figure caption)
  • Always include error bars (SEM) in figures
